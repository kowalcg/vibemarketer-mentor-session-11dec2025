## Technical Obstacles & Breakthroughs

### Building Environment
- **Reality**: Built everything on phone initially
- No desktop version of VibeCode available yet
- Workflow:
  1. Walk dog
  2. Think of idea
  3. Type prompt into VibeCode
  4. Test outside
  5. Screenshot if broken
  6. Send to ChatGPT
  7. Paste answer into VibeCode
  8. Test again
  9. Repeat 100+ times

### Major Technical Challenges

#### 1. GPS Tracking Nightmare
**Problems**:
- Inconsistent GPS signals
- Polylines not drawing correctly
- Map not updating
- Losing signal when phone in pocket
- Permissions issues
- Background updates
- Accuracy drift

**Testing Process**:
- Running outside dozens of times
- Checking if line draws
- Taking screenshots
- Debugging with ChatGPT
- Trying patches
- Retesting
- Hundreds of iterations

**Breakthrough**:
- Stuck for 2-3 weeks on pocket signal issue
- Reached out to VibeCode support
- Co-founder Ansh personally helped
- Worked together until solved

#### 2. AI Hallucinations
**Problems**:
- AI would change things not requested
- Break working code
- Overwrite logic randomly
- Complete app changes unexpectedly

**Solutions**:
- Undo changes frequently
- Revert versions often
- Sometimes rolled back hours
- Sometimes rolled back days
- **Longest rollback**: 5 full days of work

#### 3. Map System Rebuild
**Problem Discovered**:
- After a month, found mixed map systems
- Some screens: Apple Maps
- Other screens: Google Maps
- Google Maps had billing issues
- Apple Maps lacked trail coverage
- Neither solved runner needs

**Solution**:
- Complete rebuild required
- Scrapped entire map system
- Rebuilt UI
- Replaced with open-source maps
- Redid GPS logic
- Redid routing
- Redid rendering

#### 4. Learning Curve - Terminal & Tools
**Foreign Concepts**:
- What Expo was
- How to view code on device
- How to export builds
- How to submit to App Store
- What TestFlight is
- How to run build pipeline
- How to handle environment keys
- How APIs work
- Dependency conflicts
- Navigation stacks

**Learning Method**:
- Ask AI
- Read response
- Try it
- Fail
- Ask again
- Try again
- Repeat until success

## 5-Step Framework

### How to Build Your First AI App

#### Step 1: Start with a Real Pain Point
- Must be something you personally feel
- Don't choose an idea you don't care about
- Use your own frustration as fuel
- Example: Greg's spreadsheet frustration with shoe tracking

#### Step 2: Use AI to Generate PRD + Prompts
**Process**:
- Talk through idea with ChatGPT/Claude
- Request Product Requirements Document (PRD)
- Ask for feature lists
- Request user flows
- Get wireframe suggestions
- Generate prompts for VibeCode

**Tools**: ChatGPT, Claude, Gemini

#### Step 3: Build Small Pieces â†’ Test Immediately
- Don't build everything at once
- Build one tiny feature
- Test in real world
- Get feedback
- Iterate
- Example: Greg testing GPS by running outside 100+ times

**Critical**: Test outside the development environment

#### Step 4: Use Cursor + Claude Code for Debugging
**When Things Break**:
- Take screenshot
- Send to Claude/ChatGPT
- Get explanation
- Use Cursor to implement fix
- Test again

**Reality**: Expect to go back and forth hundreds of times

#### Step 5: Track Versions - AI Drift is Real
**Best Practices**:
- Save versions frequently
- Be ready to undo changes
- AI will hallucinate
- AI will break working code
- Have rollback strategy
- Keep clean version history

**Example**: Greg rolled back 5 full days of work when AI broke too much
