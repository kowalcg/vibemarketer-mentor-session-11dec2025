## Frequently Asked Questions

### How long did each app take?

**RunMate Pro:** ~8-10 weeks

### What was the hardest part?

**Top Challenges:**

1. **GPS Tracking (3 weeks stuck)**
   - Background GPS tracking is complex
   - Phone in pocket = different signal conditions
   - Required 100+ outdoor tests
   - Ansh from VibeCode personally helped fix it

2. **App Store Rejections (Emotional toll)**
   - 39 rejections for RunMate Pro
   - Each rejection required fixes
   - Nearly gave up end of August
   - Emotional exhaustion

3. **Learning Terminal/Expo/TestFlight**
   - First time seeing Terminal
   - Didn't know Expo workflow
   - TestFlight process was confusing
   - Dependencies and navigation stacks were new

**Key Learning:**
> The hardest parts were technical challenges and emotional resilience. Persistence through obstacles is essential.

## Technical Questions

### Do I need to know how to code?

**Short Answer:** No.

**Long Answer:**
- AI tools handle most coding
- You need to understand concepts, not write code
- AI teaches you as you go
- Learning happens through building

**What You Actually Need:**
- Understanding of what you want to build
- Ability to describe problems clearly
- Willingness to learn
- Patience with debugging

### What if I get stuck?

**Options:**

1. **Use AI Tools**
   - Explain problem to ChatGPT/Claude
   - Get step-by-step solutions
   - Learn as you go

2. **Ask Community**
   - Vibe Marketer Community
   - VibeCode support
   - Other builders

3. **Take a Break**
   - Fresh perspective helps
   - Don't force solutions
   - Come back with clear mind

4. **Simplify**
   - Break problem into smaller pieces
   - Solve one part at a time
   - Build up to full solution

**Key Learning:**
> Getting stuck is normal. Use resources available. Don't struggle alone.

### How do I handle App Store rejections?

**Process:**

1. **Read rejection carefully**
   - Understand what Apple wants
   - Note specific requirements
   - Don't take it personally

2. **Use AI to understand**
   - Copy rejection message
   - Paste into Cursor/ChatGPT
   - Get explanation and solution

3. **Implement fixes**
   - Make required changes
   - Test thoroughly
   - Verify compliance

4. **Resubmit**
   - Create new build
   - Submit again
   - Learn from feedback

**Key Learning:**
> Each rejection makes the app better. Apple's feedback improves the final product.
