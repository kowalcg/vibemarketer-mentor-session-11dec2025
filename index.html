The mission: Keep runners healthy and injury-free. Not just sell tape.

5

Version, Documentation & App Store Submission

AI will hallucinate and break working code. Proper documentation and version control saves you from losing weeks of work.

üìù Document Everything:

  • README.md file: Document all logic, architecture decisions, and how things work
  • CHANGELOG.md file: Track every change, feature addition, and bug fix
  • Version-specific notes: What was changed in each version and why

üíæ Version Control Strategy:

  • Keep copies of each version: Save complete working versions before major changes
  • Version-specific releases: 1.0, 1.1, 1.2 with clear documentation
  • Rollback ready: Be able to revert to last stable version quickly
  • Test before committing: Verify everything works before saving as new version

üöÄ App Store Submission:

Two Ways to Submit:
  • Option 1 - Easy Way (VibeCode): Submit your app directly from the VibeCode app - simplest method
  • Option 2 - Manual Way (Cursor/Xcode): Learn Cursor or Xcode ‚Üí Test with TestFlight ‚Üí Use Transporter to submit to App Store Connect
  • Create app identity: Design your app icon and finalize your app name early
  • Reserve domain name: Secure a domain for your app and future website (e.g., runmatepro.com)
  • Submit early: Don't wait for "perfect" - get it in the App Store
  • Learn from rejections: Apple's feedback makes your app better
  • Version tracking: Know exactly what code is live vs. in development
  • Iterate based on feedback: Real users will show you what matters

My example: I rolled back 5 FULL DAYS of work when AI broke too much. Having documented versions saved me from starting over completely.
